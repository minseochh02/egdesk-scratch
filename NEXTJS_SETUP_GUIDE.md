# Next.js Project Setup Guide for EGDesk

## Overview

This guide documents the standard EGDesk Next.js project setup, based on the reference implementation at `/Users/minseocha/Desktop/projects/youngilonc`.

## Standard File Structure

When EGDesk initializes a new Next.js project or starts an existing one, the following files should be generated/configured:

```
project-root/
├── egdesk.config.ts              # Table definitions and API credentials
├── egdesk-helpers.ts             # Database helper functions
├── next.config.ts                # Configured with dynamic basePath
├── .env.local                    # Environment variables
├── EGDESK-README.md             # Developer guide (added at runtime)
├── next.config.ts.egdesk-backup # Backup of original config
└── src/
    ├── proxy.ts                  # Database proxy middleware (Next.js 16+)
    └── lib/
        └── api.ts                # Client-side fetch wrapper
```

## Key Files and Their Purpose

### 1. `egdesk.config.ts`
Type-safe configuration file with table definitions:

```typescript
export const EGDESK_CONFIG = {
  apiUrl: 'http://localhost:8080',
  apiKey: 'your-api-key',
} as const;

export const TABLES = {
  table1: {
    name: 'ledger',
    displayName: '계장별원장',
    rowCount: 6685,
    columnCount: 16,
    columns: ['id', '일자_no_', ...]
  },
  // ... more tables
} as const;
```

**Generated by**: `@egdesk/next-api-plugin` via `npx egdesk-next-setup`

### 2. `egdesk-helpers.ts`
Database access helper functions with smart server/client routing:

```typescript
import { EGDESK_CONFIG } from './egdesk.config';

export async function callUserDataTool(toolName: string, args: Record<string, any> = {}) {
  const isServer = typeof window === 'undefined';

  if (isServer) {
    // Server-side: Call EGDesk API directly
    response = await fetch(`${apiUrl}/user-data/tools/call`, { ... });
  } else {
    // Client-side: Use proxy for CORS and tunnel support
    response = await fetch('/__user_data_proxy', { ... });
  }
  // ...
}

export async function executeSQL(query: string) { ... }
export async function queryTable(tableName: string, options?: { ... }) { ... }
export async function listTables() { ... }
```

**Key Feature**: Automatically routes to direct API on server, proxy on client.

**Generated by**: `@egdesk/next-api-plugin` via `npx egdesk-next-setup`

### 3. `src/proxy.ts`
Next.js 16+ proxy for database access (replaces middleware.ts):

```typescript
export async function proxy(request: NextRequest) {
  if (pathname.includes('__user_data_proxy')) {
    // Forward to http://localhost:8080/user-data/tools/call
    const response = await fetch(`${apiUrl}/user-data/tools/call`, {
      method: 'POST',
      headers: { 'X-Api-Key': apiKey, ... },
      body: await request.text(),
    });
    return NextResponse.json(await response.json());
  }
  return NextResponse.next();
}
```

**Purpose**: Intercepts `/__user_data_proxy` requests from browser and forwards to EGDesk MCP server.

**Generated by**: `@egdesk/next-api-plugin` via `npx egdesk-next-setup`

### 4. `src/lib/api.ts` ⚠️ **CRITICAL FOR TUNNELING**
Client-side fetch wrapper that handles basePath:

```typescript
const basePath = process.env.NEXT_PUBLIC_EGDESK_BASE_PATH || '';

export async function apiFetch(path: string, options?: RequestInit) {
  // Automatically prepend basePath to relative URLs
  const url = path.startsWith('/') && !path.startsWith('//')
    ? `${basePath}${path}`
    : path;
  return fetch(url, options);
}
```

**Why This Is Critical**:
- Next.js `basePath` does NOT automatically apply to client-side `fetch()` calls
- This is a documented Next.js limitation
- Without this wrapper, API calls from the browser will return 404 in tunneled environments

**Usage**:
```typescript
// ❌ Wrong - will fail in tunneled environment
fetch('/api/transactions')

// ✅ Correct - automatically handles basePath
import { apiFetch } from '@/lib/api';
apiFetch('/api/transactions')
```

**Generated by**: `@egdesk/next-api-plugin` via `npx egdesk-next-setup` (NEW - just added)

### 5. `next.config.ts`
Configured to read basePath from environment:

```typescript
const nextConfig: NextConfig = {
  basePath: process.env.EGDESK_BASE_PATH || '',
  assetPrefix: process.env.EGDESK_BASE_PATH || '',
};
```

**Modified by**: `dev-server-manager.ts` → `configureNextJsBasePath()`

### 6. `.env.local`
Environment variables for both server and client:

```env
# EGDesk User Data Configuration
NEXT_PUBLIC_EGDESK_API_URL=http://localhost:8080
NEXT_PUBLIC_EGDESK_API_KEY=10d8fc35-eb53-4f54-aa59-dd00f4132b82

# Tunnel configuration (set at runtime by dev-server-manager)
EGDESK_BASE_PATH=/t/tunnel-id/p/project-name
NEXT_PUBLIC_EGDESK_BASE_PATH=/t/tunnel-id/p/project-name
```

**Generated by**: `@egdesk/next-api-plugin` initially, updated by `dev-server-manager.ts` at runtime

### 7. `EGDESK-README.md`
Developer guide explaining the basePath limitation and best practices.

**Generated by**: `dev-server-manager.ts` → `writeEGDeskReadme()`

## Setup Process Flow

### When Starting a Dev Server

1. **Check if folder is empty** (`dev-server-manager.ts:1117-1120`)
   - If empty: Run `initializeNextJsProject()` to create Next.js app with `create-next-app`

2. **Install dependencies** (if needed)
   - Run `npm install`

3. **Ensure Next.js plugin is installed** (`ensureNextApiPlugin()`)
   - Install `@egdesk/next-api-plugin` from local path
   - Run `npx egdesk-next-setup` which generates:
     - ✅ `egdesk.config.ts`
     - ✅ `egdesk-helpers.ts`
     - ✅ `src/proxy.ts`
     - ✅ `src/lib/api.ts` (NEW)
     - ✅ `.env.local`

4. **Configure basePath for tunneling** (`configureNextJsBasePath()`)
   - Backup `next.config.ts` to `next.config.ts.egdesk-backup`
   - Inject `basePath: process.env.EGDESK_BASE_PATH`
   - Generate `EGDESK-README.md`

5. **Start dev server**
   - Set environment variables:
     - `EGDESK_BASE_PATH=/t/{tunnelId}/p/{projectName}`
     - `NEXT_PUBLIC_EGDESK_BASE_PATH=/t/{tunnelId}/p/{projectName}`
   - Run `npm run dev -- -p {port}`

### When Stopping a Dev Server

1. **Restore original config** (`restoreNextConfig()`)
   - Restore `next.config.ts` from backup
   - Delete backup file

2. **Remove generated docs** (`removeEGDeskReadme()`)
   - Delete `EGDESK-README.md`

## Best Practices for Developers

1. **Always use `apiFetch()` for API routes**
   ```typescript
   import { apiFetch } from '@/lib/api';

   // In components/pages
   const response = await apiFetch('/api/transactions');
   ```

2. **Use helpers for database access**
   ```typescript
   import { executeSQL, queryTable } from '@/egdesk-helpers';
   import { TABLES } from '@/egdesk.config';

   // In API routes or server components
   const data = await queryTable(TABLES.table1.name, { limit: 10 });
   ```

3. **Never use bare `fetch()` for relative URLs**
   - ❌ `fetch('/api/users')` will fail in tunneled environment
   - ✅ `apiFetch('/api/users')` handles basePath automatically
