  # Browser Recorder Import/Export Feature                                  
                                                                            
  ## Overview                                                               
  Add import/export functionality to the browser recorder, allowing users   
  to export individual test files or all tests as a bundle, and import      
  tests (single or bundles) with full chain metadata preservation. This     
  mirrors the successful pattern used in the userdata feature.              
                                                                            
  ## File Format                                                            
                                                                            
  ### Individual Test Export                                                
  - Single `.spec.js` file                                                  
  - Chain metadata embedded as comment header (if part of a chain)          
                                                                            
  ### Bundle Export                                                         
  - `.zip` archive containing:                                              
  - `tests/` folder with all `.spec.js` files                               
  - `downloads/` folder with downloaded files organized by script name      
  - `metadata.json` with export info, chain metadata, and download folder   
  info                                                                      
                                                                            
  ### metadata.json Structure                                               
  ```json                                                                   
  {                                                                         
  "exportVersion": "1.0",                                                   
  "exportedAt": "2026-03-06T10:30:00Z",

  "files": ["data.xlsx", "report.pdf"],                                     
  "totalSize": 2048576

  },                                                                        
  "chains": [                                                               
  {                                                                         
  "chainId": "chain-123",                                                   
  "scripts": [

  "scriptPath": "tests/download-step.spec.js",                              
  "scriptName": "download-step.spec.js",

  ## Implementation Steps                                                   
                                                                            
  ### 1. Backend IPC Handlers (chrome-handlers.ts)                          
                                                                            
  **Location**: Add after line 2990                                         
                                                                            
  #### Handler 1: `browser-recorder:export-single`                          
  - **Input**: `{ testPath: string }`                                       
  - **Behavior**:                                                           
  1. Validate test file exists                                              
  2. Show save dialog (default name: `{testName}.spec.js`)                  
  3. Read test file content                                                 
  4. If part of chain, prepend chain metadata as comment                    
  5. Write to selected location                                             
  - **Response**: `{ success: boolean, data?: { filePath, testName },       
  error?: string }`                                                         
                                                                            
  #### Handler 2: `browser-recorder:export-all`                             
  - **Input**: None                                                         
  - **Behavior**:                                                           
  1. Get all `.spec.js` files from output directory                         
  2. Return error if no tests found                                         
  3. Show save dialog (default: `browser-recorder-export-{date}.zip`)       
  4. Create ZIP using `adm-zip` (already in package.json)                   
  5. Add all test files to `tests/` folder                                  
  6. **For each test, check for download folder**:                          
  - Location: `~/Downloads/EGDesk-Browser/{scriptName}/`                    
  - Extract script name from filename (remove timestamp)                    
  - If folder exists, add all files to `downloads/{scriptName}/` in         
  ZIP                                                                       
  - Record folder info in downloads metadata                                
  7. Get chain metadata from `ChainMetadataStore`                           
  8. Transform chain scriptPath from absolute to relative                   
  9. Create `metadata.json` with chain metadata and download info           
  10. Write ZIP file                                                        
  - **Response**: `{ success: boolean, data?: { filePath, testCount,        
  chainCount, downloadFolders }, error?: string }`                          
                                                                            
  #### Handler 3: `browser-recorder:import-tests`                           
  - **Input**: None (uses file picker dialog)                               
  - **Behavior**:                                                           
  1. Show open dialog accepting `.zip` and `.js` files                      
  2. **If .js file**:                                                       
  - Check for existing file                                                 
  - Prompt for overwrite if exists                                          
  - Copy file to output directory                                           
  3. **If .zip file**:                                                      
  - Extract and validate `metadata.json`                                    
  - Get list of test files                                                  
  - Check for existing files                                                
  - Prompt user: "Overwrite" / "Skip Existing" / "Cancel"                   
  - Import test files based on choice                                       
  - Create mapping: old path ‚Üí new path (track old script name ‚Üí new        
  script name)                                                              
  - **Restore download folders**:                                           
  - For each test in downloads metadata, check if test was imported         
  - Extract script name from new test filename                              
  - Create download folder at                                               
  `~/Downloads/EGDesk-Browser/{newScriptName}/`                             
  - Copy all files from `downloads/{oldScriptName}/` in ZIP to new          
  folder                                                                    
  - Track restored download folders count                                   
  - Restore chains:                                                         
  - Generate new chain IDs                                                  
  - Validate all chain scripts imported                                     
  - Add scripts with updated paths                                          
  - Skip incomplete chains with warning                                     
  - **Response**: `{ success: boolean, data?: { imported, skipped, chains,  
  downloadFolders }, error?: string }`

  ## Validation Rules                                                       
                                                                            
  ### Export                                                                
  - File must exist                                                         
  - Read permissions verified                                               
  - Warn if archive >100MB                                                  
                                                                            
  ### Import                                                                
  - File extension must be `.js` or `.zip`                                  
  - ZIP must contain `tests/` folder or test files at root                  
  - Test files must end with `.spec.js`                                     
  - Metadata schema validation (if present)                                 
  - Chain integrity: all scripts must be in archive                         
                                                                            
  ### Chain Restoration                                                     
  - ‚úÖ Restored if: all scripts present, valid metadata, required fields    
  exist                                                                     
  - ‚ĚĆ Skipped if: any script missing, metadata corrupted, duplicate chain  
                                                                            
  ## Error Handling                                                         
                                                                            
  | Scenario | Message | Action |                                           
  |----------|---------|--------|                                           
  | No tests to export | "No tests available to export" | Alert, no action  
  |                                                                         
  | File not found | "Test file not found: {name}" | Alert, log error |     
  | Permission denied | "Cannot write to selected location" | Re-show       
  dialog |                                                                  
  | Invalid import file | "Invalid file format (.spec.js or .zip only)" |   
  Alert, can retry |                                                        
  | Corrupted ZIP | "Archive is corrupted or incomplete" | Alert, reject |  
  | All duplicates | "All tests already exist" | Alert when skip all |      
  | Partial success | "Imported 10, skipped 3, restored 1 chain, 5 download 
  folders" | Success with details |                                         
  | Download folder not readable | "Cannot read download folder: {path}" |  
  Log warning, continue export |                                            
  | Download folder creation failed | "Cannot create download folder:       
  {path}" | Log warning, continue import |                                  
  | Missing downloads in ZIP | "Expected downloads/ folder not found in     
  archive" | Info, continue without downloads |
